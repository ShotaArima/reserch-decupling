# 実験全体の問題設定（5シナリオ共通）

この文書は、5シナリオを読む前に **「この研究で何を1サンプルとみなして、何を当てたいのか」** を揃えるための定義集です。  
「動いた」だけで終わらないように、解釈時の注意点も明示します。

---

## 1. 1サンプルの定義

本リポジトリ実装では、FreshRetailNet-50K の `train` split から読み込んだ DataFrame の **1行を1サンプル** として扱います。

- 実装上、`load_freshretail_dataframe()` で1行=1レコードを読み込み。  
- 各特徴量セルが配列/リストの場合は、`coerce_numeric_columns()` で **平均値に潰してスカラー化** してから学習に使います。

> 重要: 時系列モデルというより、現状は「行単位の表形式学習（+ horizon 分のシフト）」です。厳密な sequence-to-sequence ではありません。

---

## 2. 入力値・global/local・目的変数・評価指標

### 2.1 入力値

- シナリオごとに `FEATURES` が異なります（詳細は各シナリオ文書）。
- どのシナリオでも、学習前に:
  1. 必要列を数値化（配列は平均）
  2. 欠損行を除外
  3. 列ごと z-score 正規化

### 2.2 global / local とは何か

`DecouplingAutoEncoder` は同じ入力 `x` から2種類の潜在表現を作ります。

- **local**: `local_encoder(x)` の出力（16次元）
- **global**: `global_encoder(x)` の出力（16次元）
- **復元**: `decoder([local, global])`

現状の実装では、
- 「localは短期変動のみ」「globalは不変要因のみ」を **厳密に保証する制約は未導入** です。  
- したがって global/local は「役割仮説」であり、Scenario 1/5 で妥当性を探る段階です。

### 2.3 目的変数（タスク別）

- Scenario 1: 入力再構成（`x -> x`）
- Scenario 2: `sale_amount` の将来値（実装は `HORIZON=7` ぶんシフトした1ステップ予測）
- Scenario 3: `hours_sale` 再構成（stockout 区間を低重み）
- Scenario 4:
  - Stage 1: 入力再構成（Recovery）
  - Stage 2: Stage 1 で得た潜在から `sale_amount` 近傍指標を予測
- Scenario 5: 目的変数というより反実仮想の挙動確認（平均 sale の差）

### 2.4 評価指標

- 回帰誤差: `MSE` / `L1`
- 予測・復元評価: `WAPE`, `WPE`
  - `WAPE = sum(|y_true - y_pred|) / sum(|y_true|)`
  - `WPE = sum(y_pred - y_true) / sum(|y_true|)`

## 3. データ構造（現実装）

### 3.1 テンソル形状

- 入力 `x`: `(N, F)`（N=サンプル数, F=特徴量数）
- local: `(N, 16)`
- global: `(N, 16)`
- 再構成 `rec`: `(N, F)`
- 予測 `pred`: `(N, 1)`

### 3.2 時系列としての扱い

- Scenario 2 は `arr[:-HORIZON]` と `target[HORIZON:]` を対応させるため、
  **HORIZON先を想定したシフト学習** になっています。
- ただし、過去窓（window）を明示した入力テンソル `(N, T, F)` はまだ使っていません。

---

## 4. 「共通と固有の分離」が意味するもの

この研究で意図する分離は次の通りです。

- **共通（global）**: 店舗/商品/需要基盤など、比較的ゆっくり変化する因子
- **固有（local）**: 天候・販促・休日・在庫状態など、時点依存の因子

ただし現段階では、これは **理論上の意味付け** であり、実装上は「2本のエンコーダに分けている」ことが中心です。  
よって主張の強さは以下に留めます。

1. 分離表現は計算上学習できる（Scenario 1）。
2. 下流タスクにつなげられる（Scenario 2/4）。
3. 交換実験で挙動差を観察できる（Scenario 5）。

---

## 5. Scenario 2 と Scenario 4 の違い（重要）

見た目が近いですが、問いが異なります。

- **Scenario 2（直予測）**  
  問い: 「raw sales をそのまま予測しても最低限機能するか？」  
  特徴: 単段、実装が簡潔、比較のベースラインに向く。

- **Scenario 4（2段パイプライン）**  
  問い: 「先に recovery を挟む設計は研究ストーリーとして成立するか？」  
  特徴: Stage 1/2 を分離し、将来は recovery 改良の寄与を検証しやすい。

研究としては、最終的に以下を示す必要があります。

- `Scenario 4 - Scenario 2` の改善量（WAPE差など）
- その改善が stockout やプロモーション条件で一貫するか
- 追加計算コストに見合うか

---

## 6. 「動いてすごいね」で終わらせないための最低限チェック

教授レビュー向けには、少なくとも次を表にして提示してください。

1. **データ分割の厳密性**: 時間リークがない split か
2. **比較対象**: raw 直予測（S2）・2段（S4）・分離なしモデル
3. **統計的頑健性**: seed 複数回の平均±分散
4. **失敗分析**: 在庫欠品日・販促日・通常日の誤差分解
5. **反証可能性**: local/global を潰した場合に性能が落ちるか

この5点が揃うと、「実装デモ」から「研究結果」へ進めます。
